# 拍卖行系统（完整版）技术方案

> 状态：**v1 已落地**（2026-07-16，commit 见 multiplayer 分支；E2E 29/29 全绿）
> 范围：**仅一口价**（竞价 = 二期）+ **收藏/历史持久化 DB**（跨重连/跨设备保留）
> 定位：补「经济流通」闭环——此前商城/公会商店都是 NPC 单向售卖，玩家间无交易。
> 前置已就位：账号/DB(SQLite·WAL)、金币体系(`PlayerWorld.gold` + `world.spendGold/addGold`)、物品 schema(`WorldItem`)、背包(`world.grantItem/removeItem`)、GameRoom intent + `worldSync` 广播范式、行会商店(`guildShopBuy`) 与商城(`mallBuy`) 权威扣费发放范式、通用滚动列表 `setupScroll`、面板接入范式(`pauseForMenu/resumeFromMenu` + `pointerdown` 守卫)。

---

## 一、核心设计原则（复用现有范式，不造轮子）

1. **全部走 GameRoom `intent`**，不新开 REST。
   理由：上架要冻结卖家背包物品、购买要改买家背包+金币，这些全是 `PlayerWorld` 内存状态。公会系统 `guildShop.ts` 注释已明确警示：走 REST 改 DB 会被玩家在 GameRoom 的内存副本覆盖（脏写）。拍卖行物品/金币流转与行会商店同质，故完全复用 `intent('guildBuy')` 范式。
2. **背包/金币改动 = 内存操作 + `worldSync` 广播**；挂单列表/搜索 = DB 查询，单独 `client.send('auctionData', list)` 下发（不走 worldSync，因为列表不是 `pw`）。
3. **卖家金币结算处理在线/离线两种情况**：
   - 在线 → 通过其 GameRoom 实例 `addGold` + `worldSync` 到账；
   - 离线 → 直接 `UPDATE characters SET world_data = json_set(...)` 改 DB JSON（安全：离线无内存副本，下次上线读到的就是加了金币的）。
   复用 `onlineClients` 注册表（`GameRoom` 模块级，好友/公会已用）查某 `charId` 是否在线。

## 二、数据层（db.ts 新增 3 表，幂等建表 + 存量迁移兜底）

```sql
-- 挂单（核心）
CREATE TABLE IF NOT EXISTS auctions (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  seller_char_id  INTEGER NOT NULL,
  seller_name     TEXT NOT NULL,
  item_data       TEXT NOT NULL,    -- JSON 序列化的 WorldItem（冻结出售物，含 id/name/type/quality/stats/enhanceLevel/refineStats/set）
  quantity        INTEGER NOT NULL DEFAULT 1,
  price           INTEGER NOT NULL, -- 一口价（金币）
  allow_bid       INTEGER NOT NULL DEFAULT 0, -- 0=仅一口价；1=接受竞价
  min_bid         INTEGER,          -- 竞价起拍价（allow_bid=1）
  highest_bid     INTEGER,          -- 当前最高出价
  highest_bidder  INTEGER,          -- 当前最高出价者 char_id
  category        TEXT NOT NULL,    -- equipment / material / consumable / title
  created_at      TEXT NOT NULL DEFAULT (datetime('now')),
  expires_at      TEXT NOT NULL     -- 挂单失效时间（created_at + 24h）
);

-- 我的收藏（可选持久化，见第十节决策2）
CREATE TABLE IF NOT EXISTS auction_favorites (
  char_id    INTEGER NOT NULL,
  auction_id INTEGER NOT NULL,
  PRIMARY KEY (char_id, auction_id)
);

-- 交易历史（可选持久化，见第十节决策2）
CREATE TABLE IF NOT EXISTS auction_history (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  auction_id      INTEGER NOT NULL,
  seller_char_id  INTEGER NOT NULL,
  buyer_char_id   INTEGER,          -- 一口价成交=买家；流拍/下架/未中标=NULL
  item_name       TEXT NOT NULL,
  price           INTEGER NOT NULL,
  kind            TEXT NOT NULL,    -- 'sold' | 'bought' | 'canceled' | 'expired' | 'bid_won' | 'bid_lost'
  created_at      TEXT NOT NULL DEFAULT (datetime('now'))
);
```

查询函数范式（复用 `db.prepare(...).get/run/all`）：
- `listAuctions(filter)`：`WHERE category=? AND price BETWEEN ? AND ? ORDER BY price ASC/DESC`，名称模糊用 `item_data LIKE '%name%'`（JSON 内 name 字段）。
- `getAuction(id)` / `myAuctions(charId)` / `myBids(charId)` / `myFavorites(charId)` / `addFavorite/removeFavorite` / `insertHistory(...)`。

## 三、服务端（server/auction.ts）

常量：
- `AUCTION_FEE_RATE = 0.05`（卖家成交手续费）
- `AUCTION_DURATION_HOURS = 24`

函数（均 `authChar` 鉴权，返回 `OpResult`）：

| 函数 | 逻辑 |
|------|------|
| `createAuction(pw, charId, itemId, quantity, price, allowBid, minBid)` | 校验物品在背包+数量足+价格>0 → `world.removeItem` 冻结 → 写 `auctions`(item_data=JSON) → `worldSync` |
| `cancelAuction(pw, charId, auctionId)` | 仅卖家+未成交+无人出价可撤 → 物品 `world.grantItem` 退回 → 删记录 → `worldSync` |
| `buyAuction(pw, charId, auctionId)` | 一口价购买：扣买家金币(`spendGold`)→ 物品发买家(`grantItem`)→ 卖家金币结算(在线/离线)→ 删挂单 → `insertHistory('sold'/'bought')` → `worldSync` |
| `bidAuction(pw, charId, auctionId, amount)` | 竞价出价（`allow_bid=1` + `amount>highest_bid` + 扣出价者金币锁币）→ 退回上一位出价者金币 → 记录 `highest_bid/bidder` |
| `listAuctions(filter)` | DB 查询返回列表（不发 worldSync） |
| `myAuctions(charId)` / `myBids(charId)` | 我的挂单 / 我参与的竞价 |
| `addFavorite/removeFavorite/myFavorites` | 收藏（决策2 决定持久化或内存） |
| `history(charId)` | 历史（决策2 决定持久化或内存） |

**竞价到期结算**（决策1 决定做不做）：
- 若含竞价：需要惰性/定时结算。方案 = GameRoom 内 `setInterval` 每 30s 扫 `expires_at < now` 且 `allow_bid=1` 的挂单 → 有出价则锁币转给卖家+物品给最高出价者（`insertHistory('bid_won'/'bid_lost')`），无出价则物品退回卖家(`expired`)。
- 若仅一口价：无需定时器，过期单由卖家手动撤或系统惰性回收（买家购买时顺带清理过期单）。

**离线卖家金币**：`addGoldToChar(charId, amt)` 封装在线/离线分支（见第一节原则3）。

## 四、客户端

### 4.1 桥接（src/systems/WorldClient.ts）
新增（复用 `sendIntent` 范式）：
```ts
export function requestAuctionList(filter: any): boolean { return sendIntent('auctionList', { filter }); }
export function requestAuctionCreate(itemId: string, qty: number, price: number, allowBid: boolean, minBid?: number): boolean { return sendIntent('auctionCreate', {...}); }
export function requestAuctionBuy(auctionId: number): boolean { return sendIntent('auctionBuy', { auctionId }); }
export function requestAuctionBid(auctionId: number, amount: number): boolean { return sendIntent('auctionBid', { auctionId, amount }); }
export function requestAuctionCancel(auctionId: number): boolean { return sendIntent('auctionCancel', { auctionId }); }
export function requestAuctionFav(auctionId: number, on: boolean): boolean { return sendIntent('auctionFav', { auctionId, on }); }
```
GameRoom 回 `auctionData` 消息 → 客户端缓存列表（像好友 `friendList` 那样存 `GameStateCore` 或模块变量）→ 面板渲染。

### 4.2 面板（src/ui/panels.ts，新增 `renderAuctionPanel(scene)`）
- 全屏遮罩 + 容器（对齐好友/公会面板），`scene.auctionPanel` 字段 + `openAuctionPanel/closeAuctionPanel`。
- **Tab 栏**：【浏览市场】【我的挂单】【收藏】【历史】。
- **搜索筛选栏**（浏览 Tab）：名称输入(HTML input) + 类型下拉(装备/材料/消耗品/称号) + 品质(白绿蓝紫金) + 价格排序(升/降) → `requestAuctionList(filter)` 重拉。
- **列表**：复用模块级 `setupScroll`（几何遮罩+滚动条+滚轮/拖拽，好友/公会/称号同款）。每行：物品名(品质色)+分类标签+价格+卖家+【购买/出价】按钮+收藏星标。
- **上架向导**：从背包选物品 + 输入数量/价格 + 勾选"接受竞价"+起拍价（决策1）→ `requestAuctionCreate`。上架后该物品从背包消失（worldSync）。
- **金币显示**：面板顶部 `金币: ${GameState.gold}`（worldSync 刷新）。

### 4.3 入口
- 快捷键 **P**（交易/市场语义；已确认 B/C/K/N/L/T/F/V/ESC/J/O/A 占用，P 空闲）。
- C 面板（属性）顶部加「拍卖行」入口按钮（仿现有「商城」入口，553 行）。

## 五、GameRoom 接线（server/rooms/GameRoom.ts）

`intent` switch 新增：
```ts
case 'auctionList': res = auction.listAuctions(data.filter); client.send('auctionData', res); break; // 不走 worldSync
case 'auctionCreate': res = auction.createAuction(pw, charId, ...); break;
case 'auctionBuy': res = auction.buyAuction(pw, charId, Number(data.auctionId)); break;
case 'auctionBid': res = auction.bidAuction(pw, charId, ...); break;
case 'auctionCancel': res = auction.cancelAuction(pw, charId, ...); break;
case 'auctionFav': ...; break;
```
涉及背包/金币改动后 `client.send('worldSync', pw)`（沿用 438 行范式）；列表/提示走 `auctionData` / `intentResult`。
`charId` 取自 `onlineClients` 注册表（好友/公会已用同一机制）。

- F.（决策1 含竞价）出价/到期结算：出价锁币→到期最高出价者得物品、卖家得币、历史记录正确

## 九、决策结论（已拍板）

> 2026-07-16 经 AskUserQuestion 拍板，两处分叉均已确定：
> 1. **竞价模式 = 仅一口价**（竞价锁币 + 到期结算定时器留作二期，DB schema 已预留 `allow_bid/min_bid/highest_bid/highest_bidder` 字段但本期不启用）。
> 2. **收藏 & 历史 = 持久化 DB**（`auction_favorites` / `auction_history` 两表，跨重连/跨设备保留；客户端 `GameState.auctionData` 仅作会话内渲染缓存）。
