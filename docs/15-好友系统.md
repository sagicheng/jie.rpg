## 0. 背景与设计取舍

《解》是多人游戏，社交关系是一等公民。此前没有任何"好友"概念（代码库零 `friend` 引用），本次为全新子系统。

设计决策（与用户确认）：

| 维度 | 选择 | 说明 |
|------|------|------|
| 添加方式 | **按角色名搜索** | 简单直接，无需好友码/推荐列表 |
| 添加流程 | **申请 → 对方同意（双向）** | 防止骚扰，双方均可见彼此 |
| 实时功能 | 在线状态 + 申请实时推送 + 上下线提醒 + **显示对方所在地图** | 全部实现 |

架构上**完全复用公会系统范式**（DB 元系统 + 实时跨房定向推送），刻意不进 `PlayerWorld`：好友关系属于社交元数据，不随角色世界存档，且实时通知必须走服务端权威推送（客户端无法伪造在线/上下线事件）。

## 1. 数据模型（DB）

`friends` 表（`server/db.ts` 建表块）：

```sql
CREATE TABLE IF NOT EXISTS friends (
  char_id INTEGER NOT NULL,           -- 本人角色 ID
  friend_char_id INTEGER NOT NULL,    -- 好友角色 ID
  status TEXT NOT NULL,               -- 'pending' | 'accepted'
  role TEXT NOT NULL,                 -- 'requester' | 'receiver' | 'friend'
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  PRIMARY KEY (char_id, friend_char_id)
);
```

**双向镜像**：任意一对好友在表中存两行（A→B 与 B→A），保证"我的好友列表"和"对方的好友列表"查询对称、且任一方移除都只影响自身视角（见 §3.4）。

关键函数（均在 `server/db.ts`）：

| 函数 | 作用 |
|------|------|
| `getCharacterByName(name)` | 按角色名解析 charId（添加入口） |
| `getFriends(charId)` | 已接受好友列表（含对方角色名） |
| `getFriendRequests(charId)` | 收到的待审申请（`role='receiver'`） |
| `getFriendship(a,b)` | 任意方向关系行（去重/存在性判断） |
| `addFriendRequest(requester, receiver)` | 事务建两行 pending（双向） |
| `acceptFriend(requester, receiver)` | 双向置为 accepted/friend |
| `removeFriend(charId, friendId)` | 删除两行（双向移除） |

## 2. 服务端 REST（`server/friends.ts`）

挂载于 Express 同端口 2567：`app.use('/api/friend', friendRoutes)`。所有接口 `authChar` 鉴权（token + charId + 账号归属，与公会同构）。

| 路由 | 入参 | 返回 | 实时副作用 |
|------|------|------|-----------|
| `POST /api/friend/add` | `charId, targetName` | `{ok, targetId, targetName}` | 若目标在线 → 推 `friendNotify{type:'request', fromCharId, fromName}` |
| `POST /api/friend/requests` | `charId` | `{ok, requests:[{charId,name}]}` | — |
| `POST /api/friend/list` | `charId` | `{ok, friends:[{charId,name,online,location}]}` | `online`/`location` 取自 `getOnlineLocation`（见 §3.1） |
| `POST /api/friend/accept` | `charId, requesterId` | `{ok}` | 若申请人在线 → 推 `friendNotify{type:'accepted', charId, name}` |
| `POST /api/friend/decline` | `charId, requesterId` | `{ok}` | 若申请人在线 → 推 `friendNotify{type:'declined', charId, name}` |
| `POST /api/friend/remove` | `charId, friendId` | `{ok}` | 若好友在线 → 推 `friendNotify{type:'removed', charId, name}` |

非法情形（均返回 `ok:false`）：未登录、角色不存在/不属当前账号、添加自己、角色名不存在、重复申请、无待审申请、非好友却移除。

## 3. 实时通知（`server/rooms/GameRoom.ts`）

复用公会系统验证过的**模块级 `onlineClients` 注册表**（sessionId → `{client, charId, name, location}`），跨房间定向投递，客户端无法伪造。

### 3.1 在线地图名

`onJoin` 时 `registerOnline(client, charId, ch.name, zoneName(pw.zone))` 把当前地图场景名写入注册表。`getOnlineLocation(charId)` 返回该玩家在线地图（不在线返回 `null`）。服务端新建 `server/zoneNames.ts`（21 区域名精简映射，避免引入客户端大文件 `Zones.ts`）。

`/api/friend/list` 据此返回 `online` 布尔 + `location` 字符串，好友面板直接展示"在线@某地图"。

### 3.2 上下线提醒

```ts
function notifyFriendsOnline(charId: number, online: boolean): void {
  const whoName = getCharacter(charId)?.name || '';
  for (const f of getFriends(charId)) {
    sendToCharId(f.charId, 'friendNotify', { type: online ? 'online' : 'offline', charId, name: whoName });
  }
}
```

`onJoin` 后 `notifyFriendsOnline(charId, true)`；`onLeave` 后 `notifyFriendsOnline(charId, false)`。向该玩家的**所有在线好友**推送上下线事件。

### 3.3 统一推送出口

`sendToCharId(charId, msgType, data)` 遍历 `onlineClients`，向匹配 charId 的所有在线会话发送（支持同账号多端）。REST 变更后调用它实现申请/接受/拒绝/移除的实时触达。

### 3.4 客户端订阅

`GameScene.ts` 在 `room.onMessage('friendNotify', ...)` 中按 `type` 路由：

| type | 行为 |
|------|------|
| `request` | 顶部飘字提示「X 申请加你为好友」；若面板开则刷新 |
| `accepted` | 飘字「X 已接受你的好友申请」；若面板开则刷新 |
| `declined` | 飘字「X 拒绝了你的好友申请」 |
| `removed` | 飘字「X 将你从好友列表移除了」；若面板开则刷新 |
| `online` | 更新 `GameState.friendOnline[charId]=true` + 飘字「X 上线了」；若面板开则刷新 |
| `offline` | 更新 `GameState.friendOnline[charId]=false` + 飘字「X 下线了」；若面板开则刷新 |

> 注意：面板打开时收到通知会 `closeFriendPanel()+openFriendPanel()` 重拉列表，确保在线态/申请实时反映；关闭时仅飘字提示。

## 4. 客户端（`src/`）

### 4.1 状态字段（`GameStateCore.ts`）

```ts
friendList: Array<{ charId: number; name: string; online: boolean; location: string }> = [];
friendRequests: Array<{ charId: number; name: string }> = [];
friendOnline: Record<number, boolean> = {};   // charId → 在线态（上下线实时维护）
```

`resetCore()` 一并清空。

### 4.2 HTTP 客户端（`src/systems/FriendClient.ts`）

仿 `GuildClient`：`authPost('/friend'+path)` 自动带 token。`add/requests/list/accept/decline/remove` 六个方法。

### 4.3 好友面板（`src/ui/panels.ts` → `renderFriendPanel`）

**K 键**开关（J=公会、G=组队）。全屏遮罩 + 容器按相机滚动偏移定位（禁用 `setScrollFactor(0)`，避免命中偏移）。左列好友列表（在线绿点 + 地图名 + 私聊/移除按钮），右列申请列表（接受/拒绝）+ 按角色名添加输入框。加载时 `Promise.all([list, requests])` 填充状态。

### 4.4 私聊入口（联动统一聊天）

面板"私聊"按钮调用 `scene.whisperTo(charId, name)`：

```ts
public whisperTo(charId: number, name?: string): void {
  this.closeFriendPanel();
  this.whisperTargetCharId = charId;
  this.switchChatChannel('whisper');
  this.focusChatInput();
  if (name) this.appendChatLine('system', '系统', 0, `正在私聊 ${name}（角色ID ${charId}），直接输入内容发送`);
}
```

`submitChat` 在 `whisper` 频道且无 `/w` 前缀时，复用 `whisperTargetCharId` 作为目标，直接输入即可发送（无需记忆对方 ID）。

### 4.5 键位与守卫

- **K**：开关好友面板（守卫同公会：非调试键、非对话/副本/战斗、确认框/组队面板/任务板优先、背包/属性面板关闭时）。
- 指针点击移动守卫、Enter 聚聊守卫均追加 `|| this.friendPanel`，面板打开时不触发移动。
- `refreshOpenPanels()` 追加好友面板重渲染，worldSync 后同步。

## 5. 测试（`scripts/test-friend.ts`）

`npm run test:friend:e2e`：起独立临时库权威服（随机空闲端口）+ 三客户端（colyseus.js）走生产链路。覆盖：

- **REST 全链路**：按名申请 → 对方申请列表 → 接受 → 双向互为好友 → 列表含在线态与地图名 → 移除（单向移除生效，双向删除验证）。
- **实时 friendNotify**：申请实时推送（target 在线即收）、接受实时推送（申请人在线收）、上下线提醒（好友进/离 game 房收 online/offline）。
- 边界：不能加自己、不存在角色被拒、重复申请被拒。

结果：**15/15 通过**。
