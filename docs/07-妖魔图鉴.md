## 一、系统定位

妖魔图鉴是《解》的**收集与策略辅助系统**（P1）。

> ⚠️ **当前状态：全部具名敌人（50种）已落地，4层级图鉴（完成领奖）已实现，称号系统已实现（2026-07-09）。** 图鉴UI面板已加入层级进度条和奖励领取。
>
> 📌 **设定（2026-07-07）**：妖魔图鉴定位为「非战斗时收集系统」，战斗中图鉴查询与弱点提示图标为非设计目标，不纳入开发。

**核心功能**：
- 记录玩家遭遇/击败过的所有妖魔信息
- 揭示妖魔的元素弱点与异常状态抗性
- 收集完成度奖励

**设计原则**：
- 图鉴 = 知识 = 力量。见过、杀过，你就更了解它。
- 不强制收集，但奖励丰厚。
- 与世界观深度绑定——每个妖魔条目附带背景笔记。

**代码实现**：
- 数据文件：`src/systems/BestiaryData.ts`
- 注册表：`NAMED_ENEMIES`（按敌人名索引）
- 查询接口：`getEnemyData()` / `getEnemyElementInfo()` / `getStatusHitRate()` / `generateNamedLoot()`

## 二、图鉴数据结构

每一条图鉴记录包含：

| 字段 | 说明 | 代码字段 |
|------|------|---------|
| **名称** | 妖魔名（唯一键） | `name` |
| **类型** | 杂妖 / 恶妖 / 妖将 / 妖王 | `type` |
| **所属区域** | 首次出现区域 | `zone`（运行时传入） |
| **元素属性** | 火/风/水/土/无 | `element` |
| **弱点** | 被克制的元素（伤害×1.5） | `weakness` |
| **抗性** | 抵抗的元素（伤害×0.5） | `resist` |
| **异常抗性** | 对12种状态的独立抗性值 | `statusResist` |
| **专属技能** | 覆盖默认模板技能 | `skills` |
| **专属掉落** | 击败后的专属掉落表 | `drops` |
| **属性倍率修正** | 覆盖默认type倍率 | `statMult` |
| **奖励倍率** | 经验/金币倍率修正 | `rewardMult` |
| **背景笔记** | 碎片化叙事文本 | `lore` |

## 三、妖魔分类与属性计算

### 3.1 层级体系

| 层级 | 战斗定位 | 出现方式 | 属性倍率（×区域PL） |
|------|---------|---------|----------|
| 杂妖 | 基础敌人，1~2回合可击杀 | 明雷、成群 | HP×5, ATK×0.8, DEF×0.3, MATK×0.5, MDEF×0.2, SPD×0.4 |
| 恶妖 | 精英敌人，需集中输出 | 明雷（大模型） | HP×10, ATK×1.2, DEF×0.5, MATK×0.8, MDEF×0.4, SPD×0.6 |
| 妖将 | 区域Boss | Boss明雷（金色光晕） | HP×25, ATK×1.8, DEF×0.8, MATK×1.2, MDEF×0.7, SPD×0.7 |
| 妖王 | 章节Boss | Boss战（大型） | HP×60, ATK×3.0, DEF×1.2, MATK×1.8, MDEF×1.0, SPD×0.9 |

> **属性计算公式**：妖魔属性 = 区域战力基数(PL) × 属性倍率 × 具名修正（结果取整）
> 例：浦原商店街（PL4）大虚·基利安(恶妖, HP×1.2修正) → HP = 4 × 10 × 1.2 = 48

### 3.2 元素属性分布（Bleach世界观）

| 元素 | 代表妖魔特征 | 主要出现区域 |
|------|------------|-------------|
| 火 | 暴躁、灼烧攻击 | 虚圈深部、无形帝国 |
| 水 | 阴冷、冻结/减速 | 流魂街、虚圈表 |
| 风 | 迅捷、多段攻击 | 空座町、静灵庭 |
| 土 | 厚重、高防御 | 润林安、地狱 |
| 无 | 基础妖魔，无属性偏向 | 空座町（杂妖为主） |

### 3.3 弱点与抗性规则

| 关系 | 伤害倍率 | 说明 |
|------|---------|------|
| 克制属性攻击 | ×1.5 | 水→火、火→风、风→土、土→水 |
| 抵抗属性攻击 | ×0.5 | 火→水、风→火、土→风、水→土 |
| 同属性攻击 | ×0.75 | 火→火、水→水等 |
| 无关属性 | ×1.0 | 魔法攻击（鬼道）对所有妖魔 |

> 代码实现：`getElementMultiplier(attackElement, targetElement, weakness, resist)`
> 玩家无element时（未选择元素共鸣）所有攻击×1.0

### 3.4 异常状态抗性（per-status独立抗性）

每个妖魔对12种异常状态有独立的抗性值（0%~100%），决定该状态是否生效。

| 抗性值 | 效果 |
|--------|------|
| 0% | 必然生效 |
| 1%~49% | 概率抵抗 |
| 50%~99% | 大概率抵抗 |
| 100% | 完全免疫 |

> 代码实现：`calcStatusHitRate(baseRate, subtype, enemyName, fallbackRes)`
> 优先查具名敌人per-status抗性表，无则fallback到`enemy.statusRes`（type默认值）
> 大部分杂妖对核心控制（冻结/眩晕）抗性30%~60%，妖将以上通常70%+或免疫

**鬼道subtype → 中文状态名映射**：

| 鬼道subtype | 中文状态名 |
|------------|-----------|
| seal/bind | 束缚 |
| slow | 减速 |
| freeze | 冻结 |
| stun | 眩晕 |
| poison | 中毒 |
| burn | 灼烧 |
| parasitic | 寄生 |
| fear | 恐惧 |
| atk_down | 攻降 |
| def_down | 防降 |
| spirit_drain | 灵消 |

## 七、图鉴层级系统 ✅ 已实现

### 7.1 四层级体系

所有敌人（含Boss）必须全部达成击杀条件，才能解锁对应层级。

| 层级 | 名称 | 条件 | 奖励 |
|------|------|------|------|
| 1 | 初级图鉴 | 全部敌人击杀×1 | 属性点+5、金币+1000 |
| 2 | 中级图鉴 | 全部敌人击杀×10 | 属性点+10、金币+5000、经验+5000 |
| 3 | 高级图鉴 | 全部敌人击杀×50 | 属性点+20、金币+20000、经验+20000 |
| 4 | 超级图鉴 | 全部敌人击杀×100 | 属性点+50、金币+100000、经验+100000 |

**判定规则**：取全部敌人中击杀次数最少的那个作为瓶颈。例如所有敌人都击杀了10次，但"地狱之王"仅击杀8次，则中级图鉴未达成。

**领取方式**：图鉴面板(N键)中每个已达成的层级旁显示`[领取]`按钮，点击即领取。

### 7.2 代码实现

- `BESTIARY_TIERS`：层级定义数组
- `getBestiaryTierReached(killedMap)`：返回最高已达成层级id
- `getBestiaryTierProgress(tierId, killedMap)`：返回指定层级的完成进度
- `GameState.claimBestiaryTierReward(tierId)`：领取层级奖励
- `GameState.bestiaryTierClaimed`：已领取层级列表

### 7.3 称号系统（已实现）

| 称号 | 条件 | 效果 |
|------|------|------|
| 新兵 | 收集5种 | 无 |
| 猎妖人 | 收集15种 | 对杂妖伤害+5% |
| 妖将杀手 | 收集25种+击败全部妖将 | 对妖将伤害+10% |
| 深渊行者 | 收集35种 | 对妖王伤害+10% |
| 斩魄刀觉醒者 | 全收集 | 全属性+5%，斩魄刀的纪念 |
