## 一、战斗总览

**类型**：经典指令式回合制（参考：《飘流幻境》）

**基本规则**：
- 遇敌后进入战斗场景（侧面视角，主角在左，敌人在右）
- 每场战斗 1~8 只敌人同屏（飘流幻境式双排站位）
- 鼠标点击选择目标敌人，独立HP条和状态
- 按速度值决定行动顺序（敌人轮流行动，全部击败才获胜）
- 每回合可选择：普攻 / 斩魄刀技能 / 鬼道 / 道具 / 防御 / 五大力量体系激活
- 全灭敌人获胜，主角HP归零则战斗失败
- 战斗开场显示 `[区域X · Y只]` 调试信息

## 二、属性体系

### 2.1 基础属性

| 属性 | 简称 | 说明 |
|------|------|------|
| **生命值** | HP | 归零则战斗失败 |
| **灵压** | MP | 技能/鬼道消耗资源 |
| **攻击力** | ATK | 物理攻击，影响普攻与物理技能伤害 |
| **防御力** | DEF | 减少受到的物理伤害 |
| **魔法攻击** | MATK | 魔法攻击，影响魔法技能与鬼道伤害 |
| **魔防** | MDEF | 减少受到的魔法伤害 |
| **速度** | SPD | 决定行动顺序，影响部分技能效果 |
| **暴击率** | CRT | 基础5%，可通过装备/技能提升 |
| **暴击伤害** | CDM | 基础150% |

### 2.2 属性成长

**升级时获得属性点，玩家手动分配。** 每级获得固定点数，各属性的加点效率受斩魄刀影响。

| 属性 | 每级获得点数 | 每点换算 |
|------|-------------|----------|
| HP | — | 1点 = +15 HP |
| MP | — | 1点 = +5 MP |
| ATK | — | 1点 = +1 ATK(物理) |
| DEF | — | 1点 = +1 DEF |
| MATK | — | 1点 = +1 MATK(魔法) |
| MDEF | — | 1点 = +1 MDEF |
| SPD | — | 1点 = +1 SPD |

**每级总点数**：6点

## 三、元素克制

**元素属性仅属于斩魄刀，人物本身无元素属性。**

### 3.1 克制关系

```
火 ──克──→ 风 ──克──→ 土 ──克──→ 水 ──克──→ 火
```

| 攻击方 | 被克方 | 伤害倍率 |
|--------|--------|----------|
| 火 | 风 | ×1.5 |
| 风 | 土 | ×1.5 |
| 土 | 水 | ×1.5 |
| 水 | 火 | ×1.5 |

**克制倍率**：克制 1.5x，抵抗 0.5x，同属性 0.75x。

### 3.2 元素在战斗中的表现

- 斩魄刀技能自带所属系别的元素属性
- 鬼道为**灵力属性**，不受元素克制影响（既不克制也不被克制）
- 敌人同样拥有元素属性，玩家需根据敌人属性调整策略

### 3.3 元素克制实现（v1.0 ✅ 已实现）

**数据来源**：`src/systems/BestiaryData.ts`

- 每个具名敌人有独立的`weakness`（弱点元素）和`resist`（抗性元素）
- 玩家的`GameState.element`（角色创建时选择的元素共鸣）决定攻击元素
- 玩家无element时（未选择），所有攻击×1.0

**接入点**：

| 行为 | 克制判定 | 代码位置 |
|------|---------|---------|
| 普攻 | 玩家element vs 敌人弱点/抗性 | `BattleScene.playerAttack()` |
| 技能（物理/魔法） | 同上 | `BattleScene.executePlayerSkill()` 伤害循环 |
| 鬼道 | 不受克制 | `executeKido()` 直接计算 |

**战斗日志提示**：伤害后显示`[克制]`（×1.5）或`[抵抗]`（×0.5）

**接口**：`getElementMultiplier(attackElement, targetElement, weakness, resist) → number`

## 四、战斗流程

### 4.1 回合结构（v3实现）

```
战斗开始 → 敌人入场动画 → 显示[区域X·Y只]
  │
  ▼
┌─────────────────────────────────────┐
│  玩家回合                          │
│  1. 显示指令菜单（攻击/技能/鬼道/道具/防御/力量体系）│
│  2. 鼠标点选目标敌人              │
│  3. 执行动作 + 特效演出           │
│  4. tickKidoStatus（状态持续-1）  │
│  5. 检查敌人是否全灭 → 胜利      │
└─────────────────────────────────────┘
  │
  ▼
┌─────────────────────────────────────┐
│  敌人回合（队列制）                │
│  1. 构建行动队列（按站位顺序）     │
│  2. tickKidoStatus（每轮仅一次）   │
│  3. 依次出队执行动作               │
│  4. 全部行动完毕 → 回到玩家回合    │
└─────────────────────────────────────┘
  │
  ▼
循环至战斗结束（全灭敌人或主角HP归零）
```

> **v3重构说明**：废弃旧版 `actedThisRound Set` + `processNextEnemy` 递归，改用 `enemyActQueue: number[]` 队列 + `enemyPhaseActive: boolean` 防重入守卫，彻底修复双重攻击bug。

### 4.2 速度排序规则

- 每回合按SPD从高到低排列行动顺序
- SPD相同时，玩家优先行动
- 速度buff/debuff实时影响当回合排序

### 4.3 多怪物战斗规则

- 敌人数量：1~8只（随区域递增，区域1-3：1-4只，区域19-21：4-8只）
- 站位：飘流幻境式双排（前排4只+后排4只），Y轴错开
- 目标选择：鼠标点击敌人精灵 → 高亮选中 → 技能/攻击指向该目标
- 敌人类型标签：普通无标签，精英显示「妖将」，Boss显示「妖王」（死亡后标签同步消失）
- 奖励结算：汇总所有敌人经验/灵子/掉落，统一发放

## 七、伤害公式

### 7.1 普攻伤害

```
物理普攻伤害 = ATK × 1.0 - DEF × 0.4
（普攻为物理攻击）
下限：不低于 ATK × 0.1
```

### 7.2 斩魄刀技能伤害

```
物理技能伤害 = ATK × 技能倍率 - DEF × 0.4
魔法技能伤害 = MATK × 技能倍率 - MDEF × 0.3
          × 元素克制倍率
          × 暴击倍率（若暴击，基础暴击率5%，暴击倍率1.5x）
          ± 随机浮动（±10%）
物理保底伤害 = ATK × 技能倍率 × 0.1
魔法保底伤害 = MATK × 技能倍率 × 0.1
```

> 物理技能特点：单体高伤、群体低伤；魔法技能特点：单体中庸、群体高伤。

**技能倍率参考**：
| 技能类型 | 倍率范围 |
|----------|----------|
| 浅打技能 | 1.2 ~ 1.5 |
| 始解技能 | 1.5 ~ 2.5 |
| 卍解奥义 | 3.0 ~ 5.0 |

### 7.3 鬼道伤害

```
鬼道伤害 = MATK × 鬼道倍率 - MDEF × 0.3
          × 暴击倍率（若暴击）
          ± 随机浮动（±10%）
（不受元素克制影响）
保底伤害 = MATK × 鬼道倍率 × 0.1
```

**鬼道倍率参考**：
| 鬼道 | 倍率 |
|------|------|
| 冲（破道之一） | 1.0 |
| 白雷（破道之四） | 1.3 |
| 苍火坠（破道之三十三） | 2.0 |
| 赤火炮（破道之三十一） | 1.5（全体） |
| 黑棺（破道之九十） | 4.0 |
| 千手皎天汰炮（破道之九十一） | 3.0（全体） |

### 7.4 治疗量

```
治疗量 = MATK × 治疗倍率 ± 随机浮动（±5%）

治：MATK × 0.8
中治：MATK × 1.2
大治：MATK × 1.0（全队）
灵复：复活 + HP回复目标最大HP的50%
结界·生：MATK × 0.3/回合（全队，持续3回合）
圣疗：MATK × 2.0（全队）
```

## 八、异常状态

### 8.1 状态列表

| 状态 | 效果 | 持续时间 | 可叠加 |
|------|------|----------|--------|
| **灼烧** | 每回合损失HP（=灵压×0.2） | 3回合 | 不可 |
| **冻结** | 无法行动 | 1~2回合 | 不可 |
| **中毒** | 每回合损失HP（=灵压×0.15） | 3回合 | 可（刷新时长） |
| **寄生** | 每回合吸血给施法者（HP×5%） | 3回合 | 不可 |
| **减速** | 速度-30% | 2~3回合 | 不可 |
| **眩晕** | 无法行动，受击后解除 | 1回合 | 不可 |
| **禁锢** | 无法行动，受击不解除 | 2回合 | 不可 |
| **嘲讽** | 强制攻击施法者 | 1~3回合 | 不可 |
| **恐惧** | 50%跳过回合 | 2回合 | 不可 |
| **降攻** | 攻击力-10%~20% | 3回合 | 可（刷新） |
| **降防** | 防御力-15% | 3回合 | 可（刷新） |
| **降灵压** | 魔法攻击-15% | 3回合 | 可（刷新） |

### 8.2 状态结算顺序

```
回合开始 → DOT伤害结算 → 状态持续回合-1 → 检查冻结/眩晕/禁锢（跳过行动）
  → 执行行动 → 回合结束
```

### 8.3 per-status异常抗性（v1.0 ✅ 已实现）

**数据来源**：`src/systems/BestiaryData.ts` → `NamedEnemyDef.statusResist`

每个具名敌人对12种异常状态有**独立的抗性值**（0~1），不再使用统一的`enemy.statusRes`。

**命中率计算**：

```
最终命中率 = clamp(0.05, 0.95, baseRate + 玩家statusAcc - 敌人该状态抗性)
```

- `baseRate`：技能/鬼道的基础命中率
- `玩家statusAcc`：来自斩魄刀成长（如疋杀地蔵+10%、逆抚+15%）
- `敌人该状态抗性`：优先查具名敌人的`statusResist[状态名]`，无则fallback到type默认`statusRes`

**鬼道subtype → 状态名映射**：

| 鬼道subtype | 状态名 | 示例鬼道 |
|------------|--------|---------|
| seal | 束缚 | 塞 |
| slow | 减速 | 这绳 |
| bind | 束缚 | 嘴突三闪 |
| freeze | 冻结 | 时间停止 |
| stun | 眩晕 | 斥 |
| poison | 中毒 | （技能机制） |

**接入点**：`BattleScene.calcStatusHitRate(baseRate, subtype, enemyName, fallbackRes)`

| 方案 | 描述 | 推荐度 |

| A. 明雷 | 地图上可见敌人，碰触触发战斗 | ★★★ |

## 十一、逃跑机制

- 普通战斗可逃跑，成功率 = (玩家SPD / 敌人平均SPD) × 50%
- Boss战不可逃跑
- 逃跑失败 = 浪费当前回合

## 十五、Boss战机制（v1.0 ✅ 已实现）

### 15.1 Boss识别

| 条件 | 判定 |
|------|------|
| Zones.ts配置 | `isBoss: true` |
| 敌人类型 | `type === '妖将' \|\| type === '妖王'` |

两者满足其一即视为Boss。

### 15.2 Boss战规则

| 规则 | 说明 |
|------|------|
| 敌人数量 | 强制1只（不被`randomEnemyCount`复制） |
| 逃跑 | 无主动逃跑指令（设计中不提供临阵脱逃）；战斗失败后可「返回据点」脱离（Boss战同理） |
| 出场演出 | 红色「⚠ BOSS战 ⚠」横幅 + 金色Boss名，总时长2.2秒 |
| 死亡 | 永久消失，不复活（普通怪30秒复活） |

### 15.3 Boss特殊机制（按Boss独立实现）

| Boss | 机制 | 触发条件 | 代码位置 |
|------|------|---------|---------|
| 大虚·亚丘卡斯 | 再生（回20%最大HP） | HP<40%，每场1次 | `processEnemyAction()` |

> 后续Boss机制（召唤小兵/阶段转换/狂暴等）待区域②~⑦落地时逐个设计

### 15.4 Boss明雷表现

- 体型1.6倍 + 金色tint(0xffcc44)
- 【BOSS】后缀标签 + 14px粗体金字
- 脉冲光晕（alpha 0.15↔0.35, scale 1.0↔1.2）
- 站桩威慑（不巡逻），呼吸缩放动画
