## 一、系统定位

地图探索是《解》的**世界呈现与交互层**（P1），承载自由探索、NPC交互、采集、遇敌、区域推进等所有非战斗场景的核心玩法。

**世界观**：Bleach（死神）——主角在空座町捡到斩魄刀，逐步觉醒力量，走过尸魂界、虚圈、完现术据点、无形帝国，最终抵达地狱深渊。

**参考基准**：《飘流幻境》——明雷遇敌、区域间自由行走、NPC交互。

**核心规则**：
- 2D俯视角八方向自由移动
- 21区域按Bleach原作世界线排列，线性链路连接（1→2→…→21）
- 区域内明雷遇敌
- 区域推进通过剧情解锁（非解放机制）

## 三、世界地图（21区域线性链路）

《解》的世界由 21 个区域按 Bleach 原作世界线**线性串联**（1→2→…→21），无大陆/海洋分割，无航海系统。区域推进通过剧情解锁，玩家抵达当前区域边界即触发切换进入下一区域。

区域名称与段位归属见 §二（Bleach世界观路线表）。线性链路示意：

```
[1 浦原商店街]→[2 空座高校]→[3 河川敷]→[4 润林安]→[5 戌吊]→[6 草鹿]
   →[7 一番队舍]→[8 技术开发局]→[9 真央灵术院]→[10 白砂原]→[11 黑腔深部]
   →[12 虚夜宫]→[13 战迹]→[14 XCUTION基地]→[15 完现术总本山]
   →[16 影之领域]→[17 星十字宫]→[18 银架城]→[19 咎人之门]→[20 无间]→[21 终焉之渊]
```

每 3 个区域对应一个力量体系段位（浅打→始解→卍解→虚化→完现术→圣文字→狱解），平滑过渡。

### 3.1 区域规模与解锁

| 项目 | 说明 |
|------|------|
| 区域总数 | 21（代码 `ZONE_CONFIGS`，id 1~21） |
| 连接方式 | 线性链路，边界检测触发切换（淡入淡出），无自由航行 |
| 解锁条件 | 剧情推进（非"解放"机制） |
| 单区域规模 | 程序化生成，含 8 NPC + 明雷妖魔 + 采集点 + 交互点 |

## 四、区域切换与快速移动

### 4.1 区域间切换

- 同幅地图内，玩家走到区域边界 → 触发淡入淡出 → 进入下一区域（代码已实现边界检测逻辑）。
- 不开放自由航行/灵舟（原「灵渊」设定的三块大陆 + 灵舟航海已废弃，见 §十五变更记录）。

### 4.2 快速移动

| 方式 | 条件 | 消耗 |
|------|------|------|
| 传送碑 → 已激活传送碑 | 目标碑已激活 | 金币×100 起 |
| 传送符 → 最近据点 | 任意 | 传送符×1 |
| 穿界门（return_point） | 各区域固定点位 | 无（返回据点） |

> 注：原「渡口 / 灵舟点对点航行」在 Bleach 线性链路模型下不再适用，已移除。

## 五、移动与交互

### 5.1 移动控制

| 控制方式 | 说明 |
|---------|------|
| 方向键 / WASD | 角色**八方向**移动 |
| 鼠标点击 | 点击地面移动至目标点 |

### 5.2 移动速度

| 地形 | 速度倍率 | 说明 |
|------|---------|------|
| 平地 / 街道 | ×1.0 | 默认 |
| 水域 / 沼泽 | ×0.6 | 减速 |
| 特殊地形（熔岩 / 虚空） | ×0.5 | 减速 + 持续伤害（设计意图，代码未实装区域环境伤害） |

### 5.3 交互系统

| 交互对象 | 触发方式 | 结果 |
|---------|---------|------|
| NPC（8类角色） | 走近 + 按键 | 对话 / 商店 / 任务 / 制造 / 强化 / 剧情 |
| 采集点 | 走近 + 按键 | 获得材料（5 分钟后刷新） |
| 宝箱 | 走近 + 按键 | 获得物品（一次性） |
| 明雷妖魔 | 碰触 | 进入战斗 |
| 穿界门 | 走近 + 交互 | 返回据点 |
| 剧情触发点 | 走进范围 | 触发剧情 |

> 每个区域固定配置 8 个 NPC（商人 / 任务 / 传送 / 制造 / 强化 / 2 支线 / 剧情），详见 §十。

## 七、明雷妖魔与遇敌

### 7.1 明雷行为

| 行为 | 说明 |
|------|------|
| 巡逻 | 固定路线上来回移动 |
| 追击 | 玩家进入警戒范围（3~5 身位），妖魔向玩家移动（速度 ×0.7） |
| 脱战 | 玩家离开追击范围后返回巡逻 |

### 7.2 遇敌规则

| 场景 | 结果 |
|------|------|
| 玩家碰触妖魔正面 | 正常开战 |
| 妖魔碰触玩家背部 | 妖魔先制攻击（先手一回合） |
| 玩家碰触妖魔背部 | 玩家先制攻击（先手一回合） |
| 逃跑成功 | 后退 3 身位，1 秒无敌 |

### 7.3 妖魔配置（代码实况）

- 每个区域在 `ZONE_CONFIGS` 中配置一组妖魔，含 `杂妖` / `恶妖` / `妖将`（Boss，`isBoss:true`）。
- 妖魔带 `element`（无 / 火 / 水 / 风 / 土 / 雷 / 光 / 暗等），用于战斗内元素克制计算。
- 刷新：离开区域后按计时恢复（设计值：杂妖 5 分钟 / 恶妖 15 分钟 / Boss 24 小时）。

## 十四、技术方向（Phaser 3 + Web）

- Tilemap：Phaser 程序化生成地图（Graphics绘制地形、装饰、采集点、敌人、NPC）
- 明雷AI：简单状态机（巡逻/追击/返回）
- 区域切换：同幅地图内边界检测触发淡入淡出切场景
- 世界地图：独立Canvas层，线性链路展示 + 迷雾遮罩
- 背包/图鉴：UI 层叠加

> **当前实现（v0.10）**：21区域Bleach世界观地图，线性链路连接，区域间边界直接切换。NPC每区8个按Bleach原著配置，商店物品按段位自动生成。明雷AI：巡逻/追击/返回。
